Minim: Programming and Simplicity

The best programming languages should be simple.

By 'simple', I mean none of the following:

  • Simple and concise syntax
  • A Simple / Small standard library
  • A simple, specific domain to target

What I really mean is this: a programming language should be a simple interface between the programmer and the computer.

I shouldn't have to spend time setting up filepaths and packages for my project that comply with specific rules. I shouldn't have to install a plethora of package managers and configure a bunch of strange dotfiles just to use somebody else's library in my project. I should be able to follow my own style standards when writing code in a programming language.

The programming language I am designing (and slowly implementing), Minim, is an attempt to be a language that complies with this definition of simplicity. To demonstrate this, here is how code in a Minim project could be organised:

Simple Project Structure

  myGameProject/
    myGameProject.min
    character.min
    monsters.min
    levels.min
    screen.min

If the programmer is comfortable with that structure, then it's fine - they can leave it that way. The program is run by interpreting the main file (myGameProject.min) as such:

minim myGameProject.min

All the other files in the folder can be accessed from within myGameProject.min simply:

monsters.goblin(); levels.loadLevel1(); etc.

...but what if the programmer wants their code split into different folders for organisation?

Folder-Based Project Structure

  myGameProject/
    myGameProject.min
    levels.min

    entity/
      character.min
      monsters.min

    graphics/
      screen.min

The files inside the other folders can be enabled for access from myGameProject.min very simply, too. All the programmer needs to do is use a with line in the main file:

  // in myGameProject.min:
  with "entity"
  with "graphics"

  //...
  screen.start();
  monsters.goblin();

There are no package declarations needed, no project structure/config file. Just folders and source code.
