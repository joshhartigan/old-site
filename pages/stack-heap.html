# The Stack and the Heap

In a typical program we will usually have some of the following things:

  • Local Variables
  • Objects
  • Instance Variables
  • Methods

Of course there are other things, but let's focus on these four. In terms of memory management we have two places for these things to live: the stack - where local variables and method calls/invocations are stored, and the heap, where all of a program's objects live.

# Local or Instance Variable?

There is a difference between local variables and instance variables which is quite easy to explain.

Local variables are variables that are declared inside a method, or function. This includes parameters. Take this example of a method written in Java:

public void power(int x, int e) {
      int result;
      for (int i = 0; i < e; i++) {
        result *= x;
      }
      return result;
}

In the power method, the parameters - x and e are local variables, as well as the variables i and result.

On the other hand, instance variables are the 'other' variables - declared inside a class, but not inside a method. They could be described as 'properties' of a class in some circumstances - as an example, a Sound class might have a volume property, as an integral instance variable. Think of instance variables as being contained in an object of the class they belong to.
