# Linked Lists

The linked list is a data structure that comes in handy when someone who is building a piece of software has a collection of data, and there is the chance that a lot of insertions and removals will need to be carried out.

In a structure such as an array, if an item is placed at the end of the list, the array's originally defined size must change. If an item was to be placed in a position that isn't the end of the list, all the items in front of it would have to be reallocated to a new position, 1 above their original index. As a side effect of this, removed items would leave wasted space that could only be removed with *more* reallocation.

However, with linked lists, every element/item/node in the list has a pointer attached to it, pointing to the next element. When a programmer wants to add an element in, rather than reallocating a lot of items, the item before the position of the item added in can have its pointer re-pointed.

An example can be shown as follows:

[a]-> [b]-> [c]-> [d]-> [e]-> [NULL / TERMINATE]

In the above diagram, each item in the linked list (represented as [value]) has a pointer 'attached' to it, that points to the next item. The last item, [e], still has a pointer, but it points to a terminating value, such as null. If [e]'s pointer pointed into empty space, the program in which the list is held would cause memory errors. Here, an item is inserted into the list:

     -> [n]-v                                       // the 'v' is a down arrow
    [a]   [b]-> [c]-> [d]-> [e]-> [NULL / TERMINATE]

it isn't a very well drawn diagram, since it has been 'drawn' in text, but what has happened is that the new element ([n]), has [a]'s pointer (that previously pointed at [b]), switched to point at [n]. As, to continue the linking of the entire list, [b] needs to be pointed to, [n]'s pointer is assigned to it.
