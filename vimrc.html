My Vim Config

Vim is a very good editor, but it can be configured to be an even better editor. Below is an annotated version of my .vimrc (the file that is loaded on Vim's opening), which can be found in its original state on my dotfiles repo.

If you are new to Vim, this isn't for you. Some very basic and/or obvious settings have been left out.

Also, my .vimrc changes a lot - therefore, the information here won't be outdated, but it will also not be the newest information at any given time.

Plugins back to top

I use Vundle to manage and install Vim plugins. These are the most notable of the plugins I use:

  Plugin 'gmarik/Vundle.vim'
  Plugin 'delimitMate.vim'
  Plugin 'ervandew/supertab'
  Plugin 'scrooloose/nerdree'

A few of these plugins are for providing a more modern look and feel to Vim: DelimitMate simply automatically closes parentheses, quotes and such.

vim-css3-syntax and vim-javascript-syntax are simply rewritten, improved, and modernised syntax highlighters for the editor.

Supertab is probably the most useful in this plugin list,Â with the exception of Vundle itself. It provides auto-completion of any word with the tab key, and retrieves words from all buffers open.

The only non-self-explanatory plugin that is left is Base 16. This is a set of 256-color color schemes for Vim which all look very good; they have become the only color schemes I use.

Navigation Keybindings back to top

Simply moving around is one of Vim's biggest strengths. I never have to use the arrow keys or the mouse - unless I want to, in which case I can.

But while all of the capabilities for fast text navigation are there, some of the things can be sped up significantly, and there is a section of my .vimr dedicated to doing this.

  nnoremap H ^
  nnoremap L $

By default, Vim's home and end keys are ^ and $, similar to characters used in regular expressions. But since these are keys that will be used often, H and L are better keys to use, with their being positioned on the home row.

As well as this, they support the fact that lowercase h and lowercase l go left and right respectively, so H and L in capital should work like an 'exaggerated' form of left and right.

  " insert mode home/end
  inoremap <c-a> <esc>I
  inoremap <c-e> <esc>A
  " command line home/end
  cnoremap <c-a> <home>
  cnoremap <c-e> <end>

Ctrl-A and Ctrl-E are the bindings for home and end in the Emacs editor. The reason I use them in Vim is because they are useful in insert mode, when the physical Home and End keys are too far away from the home row. I also use the bindings in the Vim command line for the same reason.

  " move between splits with s + hjkl, new splits with ss or sv
  nnoremap s

Most of the time I spend in Vim, I'm using a split window. When I don't have too many files open, I prefer it to tabs, as I can see everything at once. But Ctrl-W, the default Vim binding for split control, can get annoying when I need to press it a lot to get around.

The s key is on the home row, and didn't have a helpful (read: helpful for me) use previously, so I use it in place of Ctrl-W.
